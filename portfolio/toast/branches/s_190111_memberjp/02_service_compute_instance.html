Instance 소개

주요 기능

  • 원하는 하드웨어 사양을 합리적인 가격으로 이용할 수 있습니다.
  • 클릭 몇 번 만으로 빠르게 서버를 구축하고 증설할 수 있습니다.
  • 24시간 365일 무중단 서비스를 제공합니다.
  • Monitoring 서비스를 통해 인스턴스의 상태를 확인할 수 있습니다.
  • i2 Instance는 고성능 High IOPS SSD를 사용하여 빠른 I/O 성능을 보장합니다(QoS 보장).
  • u2 Instance는 저렴한 비용으로 부담 없이 이용할 수 있는 인스턴스입니다.
  • MySQL Instance에는 MySQL이 설치돼 있어 쉽게 MySQL을 사용할 수 있습니다.

서비스 대상

  • 사용량 예측이 힘든 경우
  • 프로모션, 이벤트 등 단기 트래픽 집중에 대한 해결책이 필요한 경우
  • 일정 기간 대규모 인프라(인스턴스, 스토리지)가 필요한 경우
  • 부족한 사내 인프라를 지원할 IT인프라가 필요한 경우
  • 시스템 장애를 사전에 진단하고 대응할 수 있는 관리 인프라가 필요한 경우
  • 신규 서비스 출시를 앞두고 유연한 인프라 증설이 필요한 경우
  • 가상 서버를 신속하게 확장하고 부하 분산이 필요한 경우

알림
  • Instance 요금은 네트워크 비용을 포함하지 않은 것으로, Network 과금 정책에 따라 비용이 청구됩니다.
  • Instance 요금은 스토리지 비용을 포함하지 않은 것으로, Block Storage 과금 정책에 따라 비용이 청구됩니다.
  • u2/i2 Instance 사용하려면 다음 서비스 정책을 참고하시기 바랍니다.
    • 다른 사양으로 변경할 수 없습니다.
    • Linux 계열 OS만 선택이 가능합니다.
    • 루트 볼륨을 로컬 디스크로 제공하며, 하드웨어 장애 시 데이터 복구가 불가능할 수 있습니다. 별도로 백업하거나 이중화할 것을 권장합니다.
    • Instance 생성 후에는 루트 볼륨 크기를 변경할 수 없으며, 필요할 경우 범용 HDD를 생성하여 연결할 수 있습니다.
    • u2 Instance는 u2 전용 스토리지를 루트 볼륨(OS 설치 공간)으로 사용합니다. Flavor에 따라 고정된 크기(u2.nano: 20GB, u2.micro: 20GB, u2.small: 50GB, u2.medium: 50GB, u2.large: 100GB)가 할당됩니다.
    • i2 Instance는 2018년 5월 15일 부터 신규 생성 할 수 없습니다.

인스턴스 정지 시 요금 안내 (2018년 8월 1일부터 적용)

  • 정지 상태일 때 '90% 할인된' 금액을 90일 적용하고, 90일 초과되면 정지 상태를 유지하고 정상 과금으로 전환됩니다.
  • 90%는 기존 적용되던 요금 대비 할인으로 적용됩니다. 약정 요금제를 사용중인경우 약점 요금제 기준 90%할인 적용됩니다.
  • 90일 정지 기간은 인스턴스 단위로 누적하여 적용됩니다.
  • 플로팅 IP, 로드밸런서,Block Storage,Windows라이센스등 서버에 연결된 다른 자원의 요금은 서버 정지 시에도 정상적으로 청구됩니다.
  • 불필요한 요금 청구 문제가 발생하지 않도록 사용하지 않는 리소스는 꼭 반납 처리를 해주시기 바랍니다.

알림
  • 인스턴스에서 발생하는 네트워크 전송량은 인스턴스의 하이퍼바이저 관리 툴에서 측정한 네트워크 전송량(outbound)의 총합을 기준으로 합니다.
  • 이용자 서버에서 발생하는 총 네트워크 전송량의 5% 이상 또는 20MB 이상의 국제 네트워크 전송량이 발생하면, 별도의 이용 요금을 협의하거나 국제 네트워크 전송을 제한할 수 있습니다.
  • 회사는 이용자의 네트워크 제한 시 사전에 이를 이용자에게 통보하며, 이용자가 연락처를 적지 않았거나 잘못 적어서 통보를 받지 못했을 때 발생한 피해는 보상하지 않습니다.
