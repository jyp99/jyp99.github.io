Cloud Search 소개

일반적으로 검색이란 축적되어 있는 수많은 정보 중에서 의미 있는 정보를 찾는 행위입니다. 검색은 정보 검색(information retrieval)과 DBMS 검색으로 구분할 수 있으며 아래 표와 같이 상반되는 특징으로 설명할 수 있습니다. 서비스의 특성에 따라 어떤 방식의 검색으로 구현할지 결정하지만, 정보 검색을 도입하려고 해도 서버를 구축하고 플랫폼을 개발, 운영하는 기술력을 확보하는 데 어려움이 있습니다. Cloud Search는 Console에서 몇 가지를 설정한 후 API를 호출함으로써 이러한 어려움을 해소할 수 있습니다. 또한 합리적인 비용으로 서비스의 품질과 만족도를 높이는 기회를 얻을 수 있습니다.

정보 검색과 DBMS 검색의 비교

정보 검색과 DBMS 검색의 비교
정보 검색 DBMS 검색
특징
  • 쿼리로 검색
  • 원본 문서가 비교적 비정형화된 형태가 대부분임
  • 검색어와 원본 문서의 형태소를 분석하여 섬세하게 검색 가능
  • 검색 서버를 구축하기 위한 진입 장벽이 높은 단점이 있음
  • SQL 질의문으로 검색
  • 원본 데이터가 비교적 구조적으로 정형화된 형태임
  • 검색어가 정확하게 포함되어 있는 데이터만 검색할 수 있는 단점이 있음
  • 검색 서버를 구축하고 운영하는 것보다 진입 장벽이 낮음

주요 기능

  • 국내 최초 자연어 처리 기술력으로 만든 한국어 형태소 분석기를 제공합니다.
  • 클라우드 형태로 서비스를 제공하므로 검색을 위한 별도의 서버나 플랫폼을 구축하거나 운영할 필요 없이 색인 API, 검색 API 등을 호출함으로써 어렵지 않게 우수한 품질의 검색 서비스를 구현할 수 있습니다.
  • 이중화로 서비스 안정성을 보장합니다.

서비스 대상

  • 콘텐츠를 정교하게 검색하고 싶을 때
  • 업데이트가 빈번히 일어나는 콘텐츠를 검색하고 싶을 때
  • 대량의 또는 대용량의 원본 문서라 DB 검색으로는 원하는 속도를 얻기 어려울 때
  • DBMS 쿼리를 튜닝했음에도 검색 성능이 낮을 때
  • DBMS 쿼리의 Like로는 원하는 검색 결과를 얻기 어려울 때
  • 검색 결과에서 정교한 필터링이나 정렬을 구현하고 싶을 때
  • 정확한 검색 결과로 서비스 품질을 높이고 싶을 때
  • 검색 품질을 높여 사용자의 만족도를 높이고 사용자 유입수를 늘리고 싶을 때
